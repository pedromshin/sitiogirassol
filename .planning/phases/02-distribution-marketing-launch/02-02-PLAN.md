This guide covers TWO related but distinct topics: (1) iCal technical setup and (2) the manual double-booking prevention protocol. Both are essential -- iCal alone cannot prevent double bookings due to sync delays.

The guide MUST include these sections:

**Section 1: Overview -- Why Both iCal AND Manual Protocol Are Needed**
- iCal sync has delays: Airbnb refreshes every 3 hours, VRBO every 30 minutes, Booking.com varies
- A guest can book on Booking.com, and the dates won't show as blocked on Airbnb for up to 3 hours
- The manual protocol (Section 4) supplements iCal to close this gap
- Include a simple diagram showing the hub-and-spoke sync topology with Airbnb as hub:
  ```
  Airbnb (Hub) <--iCal--> Booking.com
  Airbnb (Hub) <--iCal--> VRBO
  Airbnb (Hub) <--iCal/manual--> TemporadaLivre
  ```

**Section 2: Airbnb iCal Export URL**
- The Airbnb iCal export URL is already in the codebase: `https://www.airbnb.com/calendar/ical/1345960842338220775.ics?t=5807f23cb26b48919e07c0f8e95c05a9`
- Location in Airbnb: Listing > Availability > Link calendars > Export calendar
- This URL is what gets imported into Booking.com, VRBO, and (potentially) TemporadaLivre

**Section 3: Platform-by-Platform iCal Setup (per D-05, D-06, D-07, D-08, D-09)**

**3a. Airbnb <-> Booking.com (per D-05)**
- Export from Booking.com: Extranet > Calendar > Sync Calendars > Export Calendar > copy URL
- Import into Airbnb: Listing > Availability > Link calendars > Import calendar > paste Booking.com URL
- Import into Booking.com: Extranet > Calendar > Sync Calendars > Import > paste Airbnb iCal URL
- Use platform-default refresh intervals (per D-08) -- do not try to set custom intervals
- Sync delay: up to 3 hours (Airbnb refresh rate)

**3b. Airbnb <-> VRBO (per D-06)**
- Export from VRBO: Calendar sync > Export calendar > Copy URL
- Import into Airbnb: Listing > Availability > Link calendars > Import calendar > paste VRBO URL
- Import into VRBO: Calendar sync > Connect calendars > Import > paste Airbnb iCal URL
- Sync delay: VRBO refreshes every 30 minutes (faster than Airbnb)

**3c. Airbnb <-> TemporadaLivre (per D-07)**
- iCal support is uncertain -- test during setup
- If iCal import exists: paste Airbnb iCal URL
- If iCal export exists: copy TemporadaLivre URL and import into Airbnb
- If iCal is NOT available: use manual blocking only (Section 4 fallback)
- Note: "Even without iCal, TemporadaLivre's zero-commission model makes manual management worthwhile"

**Section 4: Double-Booking Prevention Protocol (per D-10, D-11, D-12, D-13)**

This is a step-by-step checklist the owner follows every time a booking is confirmed on ANY platform:

**Step 1: Receive Notification**
- Push notifications must be enabled on phone for ALL platforms (per D-13):
  - Airbnb: Airbnb app > Settings > Notifications > Booking confirmations ON
  - Booking.com: Pulse app > Notifications > All booking notifications ON
  - VRBO: VRBO Owner app > Settings > Notifications > Booking notifications ON
  - TemporadaLivre: Email notifications (check platform settings)
- Set distinctive notification sounds for each app so you can identify which platform booked

**Step 2: Within 1 Hour of Booking Notification (per D-10)**
Open ALL OTHER platforms and block the booked dates:
- Airbnb: Calendar > Select dates > Block
- Booking.com: Extranet > Calendar > Select dates > Close room
- VRBO: Calendar > Select dates > Block
- TemporadaLivre: Calendar > Select dates > Mark as unavailable

**Step 3: Handle Simultaneous Bookings (per D-11)**
If two bookings arrive for overlapping dates on different platforms:
- First confirmed booking wins (earlier confirmation timestamp)
- Contact the later guest immediately: apologize, explain overbooking, offer alternative dates
- Suggest rebooking for next available weekend
- Template message for this scenario (in Portuguese):
  "Ola [NOME]! Infelizmente houve um conflito de datas e esta reserva nao podera ser confirmada. Pedimos sinceras desculpas. Gostaríamos de oferecer [DATAS_ALTERNATIVAS] como opcao. Pode nos dizer se funciona?"

Create a PriceLabs setup guide at `.planning/phases/02-distribution-marketing-launch/deliverables/pricelabs-setup-guide.md`.

This guide helps the owner start a PriceLabs free trial, configure it for Sitio Girassol, and evaluate its impact after 2 weeks (per D-28, D-29, D-30).

The guide MUST include these sections:

**Section 1: What is PriceLabs and Why Use It**
- Dynamic pricing tool that adjusts nightly rates based on market demand, competitor pricing, local events, and seasonality
- Currently pricing is manual (set in Phase 1: R$280 weekday / R$380 weekend)
- PriceLabs can optimize pricing night-by-night to maximize revenue while maintaining the R$280 floor
- Cost: free 30-day trial (no credit card required), then $19.99/listing/month (~R$100/month)
- ROI justification: if PriceLabs increases ADR by even R$20/night on 10 booked nights/month = R$200/month net gain, exceeding the ~R$100/month cost

**Section 2: Registration and Airbnb Integration (per D-28)**
Step-by-step:
1. Go to pricelabs.co
2. Click "Start Free Trial" (30-day, no credit card)
3. Create account (email, password)
4. Click "+Add/Reconnect Listings"
5. Select "Airbnb" from dropdown
6. Authorize PriceLabs via Airbnb OAuth (you'll be redirected to Airbnb to approve)
7. Your Sitio Girassol listing should appear automatically
8. Click on the listing to configure pricing

**Section 3: Pricing Configuration**
Configure these settings for Sitio Girassol:

| Setting | Value | Why |
|---------|-------|-----|
| Minimum price (floor) | R$280/night | Matches Phase 1 ADR floor -- never go below this (per D-28) |
| Maximum price (ceiling) | R$700/night | Cap near holiday pricing maximum from Phase 1 |
| Base price | R$280/night | Starting point for dynamic adjustments |
| Weekend adjustment | +35% | Targets ~R$378/night (close to R$380 weekend target) |
| Cleaning fee | R$180 | Fixed, not dynamically adjusted |
| Minimum stay | 2 nights | Per LISTING-03 |

Additional settings to configure:
- Enable "Last minute discounts" only if occupancy is below 50% for next 7 days
- Disable any automatic discounts that would push below the R$280 floor
- Set "Orphan day pricing" to help fill single-night gaps between bookings

**Section 4: Understanding PriceLabs Suggestions**
- PriceLabs will show a suggested price for each night on a calendar view
- Green = suggested price is higher than your base (high demand)
- Red = suggested price is lower (but will not go below your R$280 floor)
- You can accept suggestions automatically ("Sync prices") or review manually
- Recommend: start with MANUAL review for the first 2 weeks to understand the tool's logic
- Do NOT enable auto-sync until you trust the suggestions

**Section 5: Market Data Features**
- PriceLabs shows competitor pricing in your area (Sao Roque / Sorocaba region)
- Use the "Market Dashboard" to see average rates, occupancy, and demand trends
- Compare PriceLabs suggestions against your manual Phase 1 pricing to see where the tool would charge more or less
- Look for patterns: which days/weeks does PriceLabs price higher? Those are high-demand periods you may not have identified manually.

**Section 6: 2-Week Evaluation (per D-29)**
After 2 weeks of using PriceLabs (even without auto-sync), evaluate:

| Metric | How to Check | Good Sign | Bad Sign |
|--------|-------------|-----------|----------|
| Suggested vs actual price | PriceLabs calendar | Suggestions are R$280-500 range | Suggestions consistently below R$280 floor |
| Market data quality | Market Dashboard | Shows 5+ competitors in area | Shows 0-2 competitors (limited data) |
| Booking rate | Airbnb dashboard | Same or better than before | Significantly worse (unlikely with manual control) |
| ADR trend | Compare recent bookings | ADR trending up from R$280 base | ADR stuck at floor |

Decision after 2 weeks:
- If suggestions are reasonable and market data is rich: enable auto-sync and continue to $19.99/month
- If suggestions are erratic or market data is thin: cancel trial and continue with manual pricing from Phase 1
